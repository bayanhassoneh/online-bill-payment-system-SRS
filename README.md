# online-bill-payment-system-SRS-document
A Software Requirements Specification (SRS) for an Online Bills Payment System

## Academic Context
This project was developed as a core assignment for the Software Requirements Engineering course at University. While it was a collaborative team effort , I led the project's lifecycle, including the elicitation of user and system requirements , the definition of the development methodology , and the creation of the full suite of architectural diagrams (UML & DFD)."

## 📌 Project Overview
This project provides a comprehensive **Software Requirements Specification (SRS)** and **System Design**.It allows users to pay bills via bank accounts or electronic wallets.

## Methodology
The project follows the **Waterfall Model**. This includes stages like:
*Requirements Gathering
*Design
*Development

## Key Engineering Artifacts
The documentation covers:
* **Functional Requirements:** User shall be able to pay the bills.
* **System Diagrams:**
    * UML diagrams
    * DFD Diagrams.
